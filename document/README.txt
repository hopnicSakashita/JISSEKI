■ ドキュメント構成

┌────────────────────────────────────────────────────────────────┐

└────────────────────────────────────────────────────────────────┘

【2. システム設計書.txt】
┌────────────────────────────────────────────────────────────────┐
│ 内容 : システム全体のアーキテクチャと技術仕様                   │
│ 用途 : システム構成理解、技術的な意思決定、拡張計画             │
│ 対象 : 技術者、アーキテクト、上級開発者                         │
│ 形式 : 技術仕様書、データベース設計、セキュリティ設計           │
└────────────────────────────────────────────────────────────────┘

【3. 画面詳細説明書.txt】
┌────────────────────────────────────────────────────────────────┐

│ 用途 : 機能理解、業務フロー把握、ユーザー教育                   │
│ 対象 : エンドユーザー、業務担当者、教育担当者                   │
│ 形式 : 画面別機能詳細、目的・入力項目・処理内容                 │
└────────────────────────────────────────────────────────────────┘

【4. 操作手順書.txt】
┌────────────────────────────────────────────────────────────────┐
│ 内容 : システムの基本操作方法とトラブルシューティング           │
│ 用途 : 日常業務での操作ガイド、問題解決、新人教育               │
│ 対象 : 全ユーザー、運用担当者、サポート担当者                   │
│ 形式 : ステップバイステップの操作手順、FAQ                      │

■ 利用シーン別推奨ドキュメント
────────────────────────────────────────────────────────────────

【システム導入時】
- システム設計書.txt → 全体理解
- 画面一覧.txt → 機能確認
- 操作手順書.txt → 基本操作習得

【日常業務】
- 操作手順書.txt → 操作方法確認
- 画面詳細説明書.txt → 機能詳細確認

【システム保守・拡張】
- システム設計書.txt → 技術仕様確認
- 画面一覧.txt → 影響範囲確認

【ユーザー教育】
- 画面詳細説明書.txt → 機能理解
- 操作手順書.txt → 実習ガイド

【問題発生時】
- 操作手順書.txt → トラブルシューティング
- システム設計書.txt → 技術的調査

■ システム機能サマリ
────────────────────────────────────────────────────────────────

【主要機能】
✓ データアップロード（8種類のCSV取込）
✓ 検索・照会（4種類の検索機能）
✓ 分析・レポート（12種類の分析機能）
✓ 進捗管理（7種類の進捗画面）
✓ データ入力・編集（4種類の入力機能）
✓ スライド・ダッシュボード（3種類の表示機能）

【画面総数】55画面
【API総数】23API
【主要テーブル】6テーブル（PRD_RECORD, FMC_DAT, FMP_DAT, SPC_DAT, HDC_DAT, 各種マスタ）

■ 技術情報
────────────────────────────────────────────────────────────────

【フロントエンド】
- HTML5, CSS3, JavaScript
- Bootstrap（レスポンシブデザイン）
- Chart.js（グラフ描画）
- Ajax（非同期通信）

【バックエンド】
- Python 3.x
- Flask（Webフレームワーク）
- SQLAlchemy（ORM）
- Flask-Login（認証）
- Flask-WTF（フォーム処理）

【データベース】
- MySQL
- Shift-JISエンコーディング対応
- インデックス最適化
